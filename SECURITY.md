## Handling of Reported Vulnerabilities

Once a vulnerability is reported:

1. We will confirm receipt of the report.
2. Begin an internal review and risk assessment.
3. Work on a patch or updated file(s).
4. Acknowledge the contributor (unless anonymity is requested).
5. Publish updated versions in the **Releases** section.

## Security Expectations for Users

Although this toolkit contains no executables or code with runtime security risks, users are expected to:

- Review formulas/macros in Excel files before integrating them  
- Remove sample data and replace it with actual internal data  
- Store files in secure repositories or drives  
- Avoid sharing risk registers with sensitive content publicly  
- Follow your organization's internal governance and security policies  

---

## Disclaimer

This project provides governance tooling and educational materials.  
**It does not collect, store, or transmit any user data.**  
Security risks relate only to **template misuse**, **incorrect configurations**, or **exposed sensitive content** added by users.
